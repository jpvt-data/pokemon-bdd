## 03_DECKS.ipynb — À FAIRE 🔲

**Objectif :** Base de données des meilleurs decks compétitifs humains + tier lists + génération de decks originaux.

**Plan cellules :**
```
CELL 00 — Config & imports
CELL 01 — Scraping tier lists multi-sources (4 sources croisées)
CELL 02 — Parsing + matching cartes (code_set, numero) → id_carte
CELL 03 — Export decks_competitive.csv + deck_cards.csv
CELL 04 — Historique winrates par saison + détection émergents
CELL 05 — Analyse méta (patterns, synergies récurrentes)
CELL 06 — Générateur MODE ORIGINAL (Claude API → synergies inédites)
CELL 07 — Scoring & matchup simulé vs top 5 méta (ML Python)
CELL 08 — Export deck_suggestions.csv + rapport markdown
```

**Sources tier lists (CELL 01 — croisement obligatoire) :**

| Priorité | Source | IDs cartes | Qualité |
|---|---|---|---|
| 🥇 | pokemonmeta.com | ✅ | 400 decks tournoi, daily |
| 🥇 | untapped.gg/pocket | ✅ | Winrates LimitlessTCG |
| 🥈 | ptcgpocket.gg | probable | Tournois + ranked |
| 🥈 | play.limitlesstcg.com | ✅ | Résultats tournois officiels |

**Logique matching cartes :**
- Clé = `(code_set, numero)` — seule clé fiable
- Score de confiance par deck selon nombre de sources concordantes
- Tier final = médiane pondérée 4 sources (Limitless poids x2)

**Structure decks_competitive.csv :**
```
id_deck, nom, saison_apparition, saison_active,
tier, winrate, winrate_delta_7j,
nb_tournois, meilleur_rang,
statut → [etabli / émergent / déclinant / obsolète],
archetype, type_strategie, pokemon_cles, nb_ex
```

**Structure deck_cards.csv (format long) :**
```
id_deck | id_carte | quantite | role
```
`role` = attaquant_principal / attaquant_secondaire / support / draw / disruption / energie_accel

**Génération de decks :**
- Architecture : Claude API (génération) + ML Python (scoring/validation)
- Claude API reçoit : cards_pocket_fr + méta courante + tier list + contraintes JPVT
- ML : prédiction winrate, clustering archetypes, détection cartes sous-exploitées
- MODE ORIGINAL uniquement (MODE COPIE = valeur limitée, simplifié en utilitaire)
- Coût API estimé : ~$0.006 par deck généré

**Évolution script :**
Une fois notebook stabilisé → `deck_builder.py` CLI :
```bash
python deck_builder.py --mode original --meta B3 --type Eau --contre "Mega Altaria ex"
```

## DÉCISIONS TECHNIQUES CLÉS

1. **Pas de hardcoding** — slugs pokekalos générés automatiquement, mapping dans sets_master
2. **Pas de MAJ manuelle** — toutes les sources sont scrapées automatiquement
3. **pokekalos.fr = source de vérité** pour données FR cartes (noms, effets, attaques, talents)
4. **TCGdex = source de vérité** pour noms FR sets
5. **`(code_set, numero)` = clé primaire unique** pour toutes les cartes
6. **Un deck = cartes de n'importe quelle extension** — pas de restriction mono-set
7. **Claude API + ML Python** = combo optimal pour génération decks compétitifs
8. **00_MASTER.ipynb** orchestre tout via papermill
9. **Google Sheets** = interface de lecture pour Claude
10. **Images → Drive uniquement** (copyright + poids)
